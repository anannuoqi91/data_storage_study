# US001 - 接收并校验 box_info 帧

- **角色**：R-I01 接入联调工程师
- **客户端**：接入适配端 / 存储发布核心
- **优先级**：P0

## 故事
作为一个接入联调工程师，我想把 `box_info` 帧安全送进 DBServer，并在字段异常、时间异常或设备超限时被明确拒收，以便我能稳定联调上游而不是把脏数据写进正式 release。

## 验收场景

### 场景1：正常接收
假设输入帧结构完整、时间合法、设备数未超限，
当我执行 `ingest-jsonl` 或持续接入命令时，
系统应该接收帧、更新分区状态并把数据进入待发布缓冲区。

### 场景2：异常时间拒收
假设某帧 `event_time_ms` 早于允许年代，或者明显超前当前时间，
当该帧进入系统时，
系统应该拒收整帧并把摘要写入隔离区，而不是部分入库。

### 场景3：晚到越界拒收
假设某设备已经看到更晚的时间点，
当一帧晚到超过配置窗口时，
系统应该拒收这帧并记录原因。
