# US005 - 执行聚合查询与删数治理

- **角色**：R-O03 容量治理值班
- **客户端**：受控查询端 / 运维控制端
- **优先级**：P1

## 故事
作为一个容量治理值班，我想快速看到 `obj_type` 聚合结果，并在磁盘水位高时按时间边界删掉历史分区，以便系统既能出统计结果，也能在磁盘紧张时自保。

## 验收场景

### 场景1：rollup 查询
假设我传入 `device_id + time_range`，可选 `lane_id` 与 `obj_type`，
系统应该返回按 10 分钟桶组织的 `box_count` 和 `trace_count`。

### 场景2：计划删数
假设我要先评估影响范围，
当我执行 `plan-delete`，
系统应该明确给出将删除的文件组数量和空间估算。

### 场景3：执行删数
假设保留边界需要前移，
当我执行 `delete-before` 或 `enforce-watermarks`，
系统应该切到新的 release，删除历史分区并同步清理派生数据。
