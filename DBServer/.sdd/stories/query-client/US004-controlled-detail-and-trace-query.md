# US004 - 执行受控明细与轨迹查询

- **角色**：R-Q01 轨迹分析工程师
- **客户端**：受控查询端
- **优先级**：P0

## 故事
作为一个轨迹分析工程师，我想按 `device_id + time_range` 查明细，或按 `device_id + trace_id` 查轨迹段，以便在固定 release 上稳定复盘轨迹行为。

## 验收场景

### 场景1：明细查询绑定当前 release
假设我请求某设备某时间范围的明细，
系统应该选择一个固定 release，并只在该 release 上执行查询。

### 场景2：轨迹段查询走 trace_index
假设我请求 `device_id + trace_id`，
系统应该从 `trace_index` 返回轨迹段范围，而不是要求我手动扫所有明细文件。

### 场景3：删除冲突保护
假设某个查询仍在运行，
当删数流程命中该时间范围时，
系统应该识别冲突并阻止删除直接踩到活跃查询。
