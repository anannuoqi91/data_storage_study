# 项目定义

## 一句话描述
DBServer 是一个面向 `box_info` 轨迹盒子数据的单机存储与受控查询服务，为接入侧、查询侧和运维侧提供统一的数据发布、版本切换、删数与恢复能力。

## 客户端清单

| 客户端 | 定位 | 主要用户 |
|--------|------|----------|
| 接入适配端 | 将 `DDS/cybernode` 或离线文件转换为统一帧模型并送入 DBServer | 上游接入进程、联调工程师 |
| 存储发布核心 | 负责缓冲、排序、发布、封口、删数、元数据与派生数据维护 | DBServer 进程 |
| 受控查询端 | 对明细、轨迹段和聚合查询提供固定入口 | 轨迹分析工程师、离线分析工程师 |
| 运维控制端 | 对 seal、delete、rollback、rebuild、状态查看提供受控入口 | 运维工程师、值班工程师 |
| 外部容量治理模块 | 向 DBServer 下发按时间范围删除请求 | 容量治理服务 |

## 增量结论
- 这是“已有项目加功能”模式，不需要从零设计主链路。
- 本轮主要补齐 README 已承诺但代码仍缺失的派生数据面与运维入口。
